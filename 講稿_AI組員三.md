## 開場（約 20 秒）

大家好，我是負責 AI 分析模組的組員三。

我們這個專題的核心亮點，就是讓民眾不只是「填表格」，而是上傳一張照片，系統就能自動幫你分析：這是什麼問題、有多嚴重、該怎麼處理——這整套智慧判斷的能力，就是我負責的 AI 部分。

---

## 技術架構說明（約 60 秒）

我使用的是 Google 的 **Gemini 1.5 Flash** 模型，它支援多模態輸入，也就是同時接受圖片加文字，非常適合我們的使用情境。

整個 AI 模組分成四個層次：

**第一層：圖片前處理**  
使用者上傳照片後，我用 `imageUtils.js` 在瀏覽器端自動壓縮圖片到 1280px 以內，避免傳輸過慢或 API 超時。

**第二層：Prompt 工程**  
這是最關鍵的部分。我設計了一個結構化 Prompt，明確告訴 Gemini 要用繁體中文、要輸出 JSON 格式，並且定義了四個嚴重程度的判斷標準——從「輕微」到「緊急」，對應分數 1 到 10 分。

**第三層：API 端點**  
在 Next.js 建立了 `/api/ai/analyze` 這個端點，接收前端傳來的圖片與描述，呼叫 Gemini API 後把結果回傳給前端和後端。

**第四層：前端整合**  
我封裝了 `useAIAnalysis` React Hook，讓前端組員一不需要了解 API 細節，一行 `analyze()` 就能啟動分析；同時提供 `AIResultCard` 元件直接渲染分析結果。

---

## AI 能分析什麼（約 40 秒）

給大家看一下 AI 的輸出。當民眾上傳一張道路坑洞的照片，AI 會回傳這樣的 JSON：

- **問題分類**：道路坑洞（共有八種類別）
- **嚴重程度**：HIGH，嚴重，7 分
- **30 字摘要**：「忠孝東路大型坑洞，影響行車安全」
- **詳細描述**：坑洞位置、大小、潛在危險
- **建議處理方式**：設置警示 + 填補瀝青
- **標籤**：交通安全、路面修繕
- **信心度**：92%

這份結構化資料，後端組員四可以直接存進 Supabase，地圖組員二可以根據嚴重程度決定標記顏色。

## 設計理念（約 30 秒）

我在設計 Prompt 時特別考慮了幾件事：

**穩定性**：把 Gemini 的 temperature 設成 0.2，確保每次對同一張圖的判斷不會有太大偏差。

**容錯**：Gemini 有時會在 JSON 外面加上 markdown 代碼框，我寫了解析清洗邏輯，確保一定能成功提取 JSON。

**擴充性**：如果未來要加入新的問題類別，只需修改 `CATEGORIES` 物件和 Prompt，不需要改其他程式。

## 可能被問到的問題 & 參考回答

**Q：Gemini API 要收費嗎？**  
A：Gemini 1.5 Flash 有免費方案，每分鐘 15 次請求，對我們 Demo 等級的使用量完全夠用。

**Q：如果 AI 分類錯了怎麼辦？**  
A：我在結果中有回傳 `confidence` 信心度欄位，低於 0.6 時前端可以提示使用者「AI 不確定，請確認分類」；同時系統保留使用者手動修改分類的功能。

**Q：為什麼選 Gemini 而不是 OpenAI GPT-4o？**  
A：Gemini 1.5 Flash 對多模態圖片理解效果好，而且有較慷慨的免費額度，適合學生專題。GPT-4o 雖然也很強，但費用相對較高。

**Q：圖片資料安全嗎？**  
A：圖片經過 Base64 編碼後透過 HTTPS 傳輸到 Gemini API，Google 有對應的資料隱私政策。在我們的架構中，圖片儲存是交由後端組員四用 Supabase Storage 管理，AI 分析用完即捨棄不儲存原圖。
